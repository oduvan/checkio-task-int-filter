R2HI 2.0 interface specification:
1. Data package (message) represented as a list of int numbers:
[d1, d2, d3, ..., dn], where d1 - the first letter in message
2. Each decimal number is an ASCII-code of respective character in binary + parity bit
See example:

    Chr | Dec  |   Bin   |P|  Bin + P | Dec
    ---------------------------------------
    'P' |  80  | 1010000 |0| 10100000 | 160
    'y' | 121  | 1111001 |1| 11110011 | 243
    't' | 116  | 1110100 |0| 11101000 | 232
    'h' | 104  | 1101000 |1| 11010001 | 209
    'o' | 111  | 1101111 |0| 11011110 | 222
    'n' | 110  | 1101110 |1| 11011101 | 221

        Message = [160, 243, 232, 209, 222, 221]

You have to implement R2HI 2.0 translator/analyzer that must translate received data package (list of int) to string message. Erroneous numbers must be removed from package (list). Erroneous means decimal value contains wrong (odd) number of '1' in a binary form:

'P' → 80 → 1010000 + 0 → 10100000 → ...101 10000... → 10110000 (Erroneous, 4-th bit was inverted)
Notice: The number of bits' substitutions can be any odd number (1, 3, 5, ...)

Example:
Input message: [144, 16, 210, 214]
Remove erroneous characters (binary): [10010000, 00010000, 11010010, 11010110]
Result (binary): [10010000, 11010010]
Decimal: [144, 210]
Message string: "Hi"

Input: A list of int numbers

Output: Message as a string

How it is used: Parity bits are used as the simplest form of error detecting code. They allow to detect odd number of bits' substitutions in data
